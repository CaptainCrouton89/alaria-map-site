# Lore migration plan: pins + wiki prose тЖТ flat codex entities (in-repo)

The Alaria wiki moved INTO this repo as one flat file per entity under `content/codex/entities/`.
Those files are the single source for BOTH the map and the `/codex` wiki; `data/locations.json` and the
compiled codex JSON are generated by `scripts/build-codex.mts`.

## Hard invariants (still enforced)

1. **IDs never change.** An entity's `id` (string) equals its existing pin/codex id. Every id in
   `pinned.json` maps to exactly one entity file with byte-identical `coordinates`. Verify
   programmatically, not by eye. (Known pre-existing stray: pin `797` has no entity file тАФ collapsed
   into a neighbor; not a regression.)
2. **Entity files are canonical** тАФ never hand-edit the generated `locations.json` / compiled codex.

## Two tiers of entity

Same frontmatter schema (`.claude/rules/entity-files.md`); they differ only by `coordinates`.

- **Geographic (pinnable)** тАФ `entityType` тИИ {region, city, town, fortress, ruins, wilderness, water,
  poi}; HAS `coordinates`; renders on the map and codex.
- **Non-geographic (codex-only)** тАФ race, creature, person, magic, plane, deity, daemon, faction,
  artifact, event, era, nation. NO `coordinates`; codex only.

3,018 geographic entities generated from `pinned.json` тИк `manual-pins.json` (deduped via

## тЬЕ Phase B тАФ non-geographic tier (DONE for the non-atlas trees)

Converted via **`scripts/convert-lore.mts`** (`--tree <subdir> --type <entityType> [--recursive]`),
sourcing the external wiki at `../heart-rush-tools/world-wikis/alaria/`:

| Tree | type | count |
|---|---|---|
| races | race | 73 |
| bestiary (+dragons, diseases) | creature | 180 |
| history_and_lore/dramatis_personae | person | 37 |
| history_and_lore/{artifacts,events,timelines} | artifact/event | 9 |
| magic_and_knowledge | magic | 25 |
| cosmology_and_religion | plane | 26 |
| nations_and_powers/daemons | daemon | 13 |
| nations_and_powers/factions | faction | 12 |

Conventions established by the converter:
- **Lore/mechanics split** тАФ TTRPG rules (ability blocks, `CN` saves, XP, dice) are split out of the
  lore body at a `<!-- mechanics -->` sentinel. `build-codex` emits a `mechanics` field; `CodexBody`
  renders it behind a collapsed "Game mechanics" toggle. Nested ability blocks are context-labeled
  with their parent section.
- **id namespace** `<type>-<slug>`; **filenames are `<id>.md`** (NOT name-slug) so Phase B files can
  never clobber a geographic entity's bare name-slug file. The converter refuses to overwrite a
  different id.
- Inline `Tags:` lines тЖТ frontmatter `tags`; `# `/`## ` heading тЖТ `name`; CLAUDE.md/READMEs skipped.
- **Inter-wiki links** (`[text](../path/File.md)`) are stripped to plain text everywhere (Phase A +
  B); the build's name mention-scan re-links them as real codex links. New category mappings added:
  `race`, `magic`, `daemon`.

## Remaining / future passes

- **`nations_and_powers/states` (142 docs) тАФ NOT bulk-converted.** 129/142 already exist as
  geographic entities (added via pinning). The ~12 unmatched are multi-polity abstractions (e.g.
  Aldriktch Trade Alliance, Kendrel Kingdoms, Postronamas Empire) with no single pin тАФ add as
  coordinate-less `nation`/`faction` entities if/when wanted, reconciling against existing ids/aliases
  to avoid duplicates.
- **Relations/edges for Phase B are empty.** Non-geographic entities have no `within`/typed edges yet,
  so they're invisible to containment navigation (reachable by search + category + mention-scan).
  `capitalOf`, `borders`, and other `relations` are populated by the lore passes below.
- **`weight` / `atmosphere`** тАФ mostly unset; authored during lore passes (defaults baked by
  entityType meanwhile).

## Generation order (lore passes, not migration)

regions тЖТ wilderness тЖТ capitals тЖТ waters тЖТ towns тЖТ POIs тЖТ ruins. Each pass fills body + blurb + edges
for its type, loading parent + children + bordering entities as context first. See the lore review
pass for the country-by-country relationship-authoring workflow.
